# Docker Integration

Tuti CLI manages Docker environments for local development.

## Docker Service Management

### DockerComposeService

Located in `app/Services/Docker/`:

```php
final class DockerComposeService
{
    public function start(string $composeFile): bool;
    public function stop(string $composeFile): bool;
    public function restart(string $composeFile): bool;
    public function logs(string $composeFile, string $service = null): string;
    public function status(string $composeFile): array;
}


docker-compose.yml Generation

Generated by `StackComposeBuilderService`:

1. Load base docker-compose.yml from stack
2. Apply selected service stubs
3. Merge environment-specific overrides
4. Generate final file in `.tuti/`

### Environment Variables

Generated by `StackEnvGeneratorService`:
- Load template from stack (`.env.dev.example`)
- Interpolate variables with project-specific values
- Generate `.env` in project root

## Best Practices

- Use Docker Compose for service orchestration
- Store Docker config in `.tuti/` directory
- Use environment variables for sensitive data
- Provide health checks for all services
- Use named volumes for data persistence
- Document all ports in README
